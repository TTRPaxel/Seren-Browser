Privacy & Security

Built for the paranoid.
Good for everyone.

Every feature exists to reduce what the internet can learn about you — without making the browser annoying to use.

🛡
Ad & Tracker Blocking
60+ known tracker and ad network domains are blocked before requests leave your machine. A live counter in the navbar shows you exactly how many requests have been blocked in your session.
🔗
Tracking Parameter Stripping
UTM tags, fbclid, gclid, msclkid, and 30+ other tracking query parameters are stripped from every URL before the request goes out. Sites never see where you came from.
📡
WebRTC Leak Protection
WebRTC is restricted to public interfaces only, preventing your real local IP address from leaking to websites — even when using a VPN.
Always on
🎨
Canvas Fingerprint Resistance
Per-session noise is injected into canvas pixel reads, WebGL renderer strings, and AudioContext data. Your browser fingerprint changes every restart, making cross-site tracking by fingerprint ineffective.
🌐
DNS-over-HTTPS
Encrypt your DNS queries so your ISP can't see which domains you're visiting. Choose from Cloudflare, Google, Quad9, AdGuard, or NextDNS — all free, no account required.
🔍
SearxNG Integration
Seren uses SearxNG as its default search engine — a self-hostable metasearch engine that doesn't track queries, build profiles, or sell your searches to advertisers.
🤖
Local AI Assistant — No Cloud Required
The built-in AI sidebar connects to a locally hosted Ollama instance. Your conversations, page context, and queries never leave your machine. No OpenAI. No API keys. No usage logs. If you're not running Ollama locally, the feature simply isn't there — it doesn't phone home.
Philosophy

Privacy shouldn't be
a premium feature.

Most browsers treat privacy as an opt-in. You get tracking by default, and if you care enough to dig through settings, you can disable some of it. Seren starts from the opposite position.

Every default is chosen to minimize exposure. Nothing phones home. No "anonymous" telemetry. No sync to external servers. No built-in accounts. The browser is a tool, not a platform.

Seren is built on the belief that you shouldn't need to understand browser internals to browse without being profiled. The protection is on by default, the settings are plain English, and nothing is hidden.

It's experimental software, honestly. But it's honest software.

01
Transparent by default
The blocked request counter is always visible. You can see what's being blocked and add your own rules.
02
Local-first
Search, AI, and settings all run locally or on infrastructure you control. Nothing is routed through Seren's servers — there are none.
03
No dark patterns
No "accept all cookies" popups generated by the browser itself. No nagging to create an account. No recommended content.
04
Honest about limitations
Seren is experimental and under development. It doesn't claim to be Tor Browser. It just tries to be meaningfully better than the default.
05
Configurable, not locked down
You can turn off JavaScript, switch DoH providers, or load your own extensions. Privacy is the default, not the ceiling.
Get Started

Up in three commands.

Seren runs directly from Python. No installer, no admin rights, no bloat.

1
Prerequisites
You'll need Python 3.10 or newer. A running SearxNG instance is required for search — you can use the public instance at 25.32.56.63:5000 or self-host your own.
2
Clone & install dependencies
Clone the repository and install the two required packages. PyQt6 and PyQt6-WebEngine are the only dependencies.
3
Run
Launch the browser. On first run, open Settings (⚙ or Ctrl+,) and point the SearxNG URL to your instance if you're self-hosting.
✦
Optional: AI Assistant
Install Ollama and pull a model (ollama pull llama3). The sidebar will auto-detect your models. Everything stays local.

FAQ

Questions.

Answers to things people might actually ask.

Seren makes a genuine effort — WebRTC blocking, canvas noise, tracker blocking, no telemetry. But it's not Tor Browser. Your IP is still visible to every site you visit unless you use a VPN. It's significantly better than Chrome or Edge out of the box, but it's not an anonymity tool. We try to be honest about that distinction.
The actual web rendering is done by QtWebEngine, which is Chromium under the hood — so page rendering performance is identical to Chrome. Python only handles the UI shell, settings, and privacy features. In practice you won't notice any difference.
No. There's a public SearxNG instance you can use at 25.32.56.63:5000 — just point Seren's search URL there in Settings. Self-hosting is recommended for maximum privacy since it means no one else can see your queries, but it's not required to get started.
No. The AI sidebar only connects to your local Ollama instance at localhost:11434. No requests go to OpenAI, Anthropic, or any other external service. If Ollama isn't running, the sidebar simply doesn't work — it doesn't fall back to a cloud API.
Yes — Seren supports loading unpacked Chromium extensions via the Extensions manager (⧉ button). Download the uBlock Origin Chromium MV3 build, extract the ZIP, and load the folder. Note that QtWebEngine's extension API support varies, so not all extensions will work perfectly.
No. Seren is an independent personal project. It's not affiliated with any browser vendor, search engine, AI company, or commercial entity. It's just a browser someone built because they wanted one that actually respected their privacy.
